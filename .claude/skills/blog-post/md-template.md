# 블로그 MD 템플릿

아래 형식으로 MD 파일을 작성하면 스킬이 파싱하여 블로그 글을 생성합니다.

---

```markdown
---
title: "블로그 제목"
category: "행사 기획"       # 행사 기획 | 행사 후기 | 트렌드 | 체크리스트 | 장비/기술
tags:
  - 세미나
  - 체크리스트
thumbnail: "./thumbnail.jpg"  # 로컬 파일 경로 또는 URL (선택)
excerpt: "블로그 목록과 SNS 공유에 표시될 요약문"
is_published: true

# ── SNS 링크 (선택 — 안 적으면 회사 메인 계정으로 자동 연결) ──
naver_blog_url: "https://blog.naver.com/paran-company/게시물번호"
instagram_url: "https://www.instagram.com/p/게시물ID/"
youtube_url: "https://www.youtube.com/watch?v=영상ID"

# ── SEO 필드 (선택 — 안 적으면 자동 생성) ──
seo_title: "검색 결과에 표시될 제목 (최대 60자)"
seo_description: "검색 결과에 표시될 설명 (최대 155자)"
focus_keyword: "이 글의 핵심 검색어"
slug: "직접-지정-슬러그"
---

블로그 본문을 마크다운으로 작성합니다.

## 소제목

본문 내용...

![이미지 설명](./photo.jpg)

> 인용문
```

## 필드 설명

### 프론트매터 (YAML)
| 필드 | 필수 | 설명 |
|------|------|------|
| title | O | 블로그 제목 — 페이지에 표시되는 제목 (최대 200자) |
| category | X | 카테고리 (행사 기획 / 행사 후기 / 트렌드 / 체크리스트 / 장비/기술) |
| tags | X | 태그 목록 (최대 10개, 각 30자 이내) |
| thumbnail | X | 대표 이미지 (로컬 경로 또는 URL) |
| excerpt | X | 요약문 (최대 500자, SNS 공유용) |
| is_published | X | 발행 여부 (기본 true) |

### SNS 링크 (선택 — 블로그 상세 페이지 왼쪽 사이드바에 표시)
| 필드 | 필수 | 설명 |
|------|------|------|
| naver_blog_url | X | 이 행사의 네이버 블로그 글 URL. 안 적으면 회사 메인 블로그로 연결 |
| instagram_url | X | 이 행사의 인스타그램 게시물 URL. 안 적으면 회사 메인 인스타로 연결 |
| youtube_url | X | 이 행사의 유튜브 영상 URL. 안 적으면 회사 메인 유튜브로 연결 |

### SEO 필드 (선택 — 안 적으면 본문 분석 후 자동 생성)
| 필드 | 필수 | 설명 |
|------|------|------|
| seo_title | X | 검색 결과에 표시될 제목 (최대 60자). 안 적으면 focus_keyword + title 기반 자동 생성 |
| seo_description | X | 메타 디스크립션 (최대 155자). 안 적으면 본문 첫 문단 + keyword 기반 자동 생성 |
| focus_keyword | X | 이 글이 타겟하는 핵심 검색어 (예: "기업행사 기획"). 안 적으면 본문에서 자동 추출 |
| slug | X | URL 슬러그 직접 지정. 안 적으면 focus_keyword 또는 title에서 자동 생성 |

### 자동 처리 필드
- `slug` → focus_keyword가 있으면 keyword 기반, 없으면 title에서 자동 생성
- `seo_title` → focus_keyword + title 조합 (최대 60자, "| 파란컴퍼니" 접미사)
- `seo_description` → focus_keyword 포함 + 본문 핵심 요약 (최대 155자)
- `og_image_url` → thumbnail URL 사용
- `published_at` → is_published: true일 때 현재 시간
- `is_featured`, `sort_order` → 기본값 false / 0
- `naver_blog_url`, `instagram_url`, `youtube_url` → 안 적으면 회사 메인 계정 자동 연결

### 본문
- 마크다운 문법으로 작성
- `![설명](./파일경로)` 형식의 로컬 이미지는 자동으로 Supabase Storage에 업로드 후 URL 치환
- 외부 URL 이미지는 그대로 사용
- 마크다운 → HTML 변환은 `marked` 라이브러리로 자동 처리

### 이미지 최적화
- 업로드 시 자동 처리: 최대 1200px, WebP quality 90, 최대 5MB
- 포트폴리오 현장사진 URL도 사용 가능 (이미 최적화되어 있음)

## SEO 자동 생성 예시

팀장님이 이렇게만 적으면:
```yaml
---
title: "성공적인 기업 행사를 위한 체크리스트 10가지"
category: "체크리스트"
tags:
  - 기업행사
  - 체크리스트
excerpt: "행사 D-30부터 당일까지, 놓치면 안 되는 핵심 체크리스트"
---
```

스킬이 본문을 분석해서 자동으로:
- `focus_keyword`: "기업행사 체크리스트" (본문에서 가장 많이 등장하는 핵심 키워드 조합)
- `seo_title`: "기업행사 체크리스트 10가지 | 파란컴퍼니" (60자 이내)
- `seo_description`: "기업행사 준비 D-30부터 당일까지 필수 체크리스트 10가지. 장소, 장비, 예산 등 놓치기 쉬운 항목을 정리했습니다." (155자 이내)
- `slug`: "기업행사-체크리스트" (keyword 기반, 짧고 명확)

## SEO 주의사항: 네이버 블로그와 내용 차별화

**같은 행사를 다루더라도 네이버 블로그와 자사 사이트 블로그의 내용은 반드시 다르게 작성해야 합니다.**
동일 내용을 복사하면 구글이 중복 콘텐츠로 판단하여 두 사이트 모두 검색 순위가 하락합니다.

### 차별화 방법

| | 네이버 블로그 | 자사 사이트 블로그 |
|---|---|---|
| **타겟 검색엔진** | 네이버 검색 | 구글 검색 |
| **글의 관점** | 현장 후기, 사진 중심 | 기획 프로세스, 노하우 중심 |
| **톤** | 가볍고 친근한 후기 | 전문적인 사례 소개 |
| **키워드** | 네이버 검색량 높은 키워드 | 구글 SEO 타겟 키워드 |
| **예시 제목** | "교육협력포럼 현장 스케치" | "공공기관 포럼 기획 대행 사례" |

### 규칙
- 제목(H1)과 본문 텍스트가 **70% 이상 달라야** 중복으로 판정되지 않음
- 같은 사진은 사용해도 되지만, 글의 구성과 텍스트는 다르게 작성
- 네이버 블로그: "이런 행사였어요" (후기) / 자사 사이트: "이렇게 기획했어요" (노하우)
