# 블로그 이미지 생성 스킬

## 역할

SEO 가이드 블로그 글에 들어갈 이미지를 AI로 생성한다.
포트폴리오 사진은 사용하지 않음 (홈페이지에 이미 있으므로 중복).

## 트리거

| 사용자 표현 | 동작 |
|------------|------|
| "블로그 이미지 만들어줘" | 주제/유형에 맞는 이미지 생성 |
| "블로그 썸네일 만들어줘" | Ideogram V2로 썸네일 생성 |
| "블로그 인포그래픽 만들어줘" | Flux 2 Flex로 인포그래픽 생성 |

blog-seo 스킬에서 자동 호출될 수도 있음.

---

## 모델 선택 규칙

### 사용 가능한 모델

| 모델 | MCP | 강점 | 약점 |
|------|-----|------|------|
| **Flux 2 Flex** | Replicate | 텍스트 렌더링 최강, 인포그래픽/레이아웃, 참조 이미지 10장 | 한국어 텍스트 불안정할 수 있음 |
| **Ideogram V2 Turbo** | Replicate | 텍스트/타이포그래피 특화, 포스터/로고 스타일 | 빠르지만 품질 중간 |
| **Gemini 3 Pro** | 나노바나나 | 고해상도 4K, 분위기 이미지, 한국어 이해 | 느림 (240초) |

### 이미지 유형별 모델 자동 배정

| 이미지 유형 | 모델 | 이유 |
|------------|------|------|
| 비교 인포그래픽 | **Flux 2 Flex** | "Text rendering is where Flex really shines" |
| 프로세스 플로우차트 | **Flux 2 Flex** | 복잡한 레이아웃 + 텍스트 처리 |
| 비교표/도표 카드 | **Flux 2 Flex** | 타이포그래피 + 레이아웃 |
| 체크리스트 요약 카드 | **Flux 2 Flex** | 텍스트 중심 이미지 |
| 썸네일/타이틀 카드 | **Ideogram V2** | 포스터/로고 스타일 특화 |
| 분위기 일러스트/배경 | **Gemini 3 Pro** | 고해상도, 분위기 이미지 |
| 공간 배치도/평면도 | **Flux 2 Flex** | 구조적 레이아웃 |

### 모델별 호출 방법

**Flux 2 Flex (Replicate):**
```
mcp__replicate__create_predictions
model: black-forest-labs/flux-2-flex
input: { prompt, num_inference_steps: 20, guidance_scale: 3.5 }
```

**Ideogram V2 (Replicate):**
```
mcp__replicate__create_predictions
model: ideogram-ai/ideogram-v2-turbo
input: { prompt }
```

**Gemini 3 Pro (나노바나나):**
```
mcp__nanobanana__generate_image
model: gemini-3-pro-image-preview (필수 — Flash 모델 사용 금지)
resolution: 2K
```

## 스타일 규칙

### 공통 프롬프트 접두사

```
전문적인 인포그래픽 스타일, 깔끔한 모던 디자인, 미니멀 기업 디자인, 플랫 디자인.
```

### 이미지 수량 규칙

**고정 수량 없음. 글 내용에 맞게 적절한 수만큼 생성한다.**

- 썸네일 1장은 필수 (본문 첫 번째 이미지와 동일하게 설정)
- 본문 이미지는 **내용상 도움이 되는 곳에만** 배치 (2~5장 범위)
- 억지로 수를 맞추지 않는다 — 3장으로 충분하면 3장, 5장이 필요하면 5장
- 이미지 없이도 전달되는 섹션에는 넣지 않는다

### 색상 규칙: 주제에 맞는 자유로운 색상

**팔레트에 국한하지 않는다.** 아래 팔레트는 참고용 예시일 뿐, 주제에 어울리는 색이라면 어떤 색이든 자유롭게 사용한다.

- **같은 글 안의 모든 이미지(썸네일 + 본문)** 는 동일한 색상 톤으로 통일
- **글 주제에 어울리는 색상**을 자연스럽게 선택 — 파란색 계열에 국한하지 말 것
- 블로그 목록에서 바로 옆에 나열되는 글끼리는 되도록 다른 톤
- **따뜻한 색(오렌지, 코랄, 테라코타), 자연 색(올리브, 포레스트, 모스), 뉴트럴(웜그레이, 베이지, 차콜)** 등도 적극 활용

**참고 팔레트 (예시, 이 외에도 자유롭게 사용 가능):**

| 계열 | 예시 색상 | 적합한 주제 |
|------|----------|-----------|
| 블루 | #2563eb, #0ea5e9 | 기획, 프로세스 |
| 틸/시안 | #0d9488, #06b6d4 | 장소, 장비, 현장 |
| 인디고 | #4f46e5, #6366f1 | 비교, 분석, 정보 |
| 앰버/오렌지 | #d97706, #ea580c | 비용, 견적, 예산 |
| 에메랄드/그린 | #059669, #16a34a | 체크리스트, 실무 |
| 로즈/코랄 | #e11d48, #f43f5e | 후기, 성과, 결과 |
| 바이올렛/퍼플 | #7c3aed, #a855f7 | 제작물, 콘텐츠 |
| 슬레이트/차콜 | #475569, #374151 | 입찰, 제안서 |
| 테라코타/웜 | #c2410c, #b45309 | 현장 노하우, 운영 |
| 올리브/모스 | #65a30d, #4d7c0f | 가이드, 교육 |

프롬프트에 색상을 직접 지정할 때: "메인 색상 {코드}" 또는 "따뜻한 오렌지 톤" 같은 자연어 표현도 가능

### 연도 규칙

- 이미지에 연도가 표시되는 경우 **반드시 현재 연도**를 사용할 것
- 프롬프트에 연도를 명시하지 않으면 AI가 2023, 2024 등 학습 데이터 기준 연도를 넣을 수 있음
- 연도가 포함될 수 있는 상황(달력, 일정표, 문서, 배너 등)에서는 프롬프트에 "2026년" 등 현재 연도를 명시할 것
- 연도가 불필요한 이미지에서는 아예 연도가 나오지 않도록 프롬프트에서 제외

### 텍스트 규칙 (재생성 방지를 위해 반드시 준수)

**이미지 생성 전 텍스트를 먼저 확정하고, 그에 맞게 이미지를 생성한다.**
두 번 생성하는 일이 없도록 아래 규칙을 철저히 지킨다.

- **반드시 한글로 작성** — 영어 이미지는 한국어 블로그와 어울리지 않음
- **텍스트는 제목 + 짧은 키워드만** — AI 이미지 생성기는 작은 설명 텍스트를 한글로 정확히 렌더링하지 못함
- **긴 설명문/본문 텍스트 절대 넣지 않기** — 2줄 이상의 설명은 100% 깨짐 (AI 횡설수설 현상)
- **프롬프트에 이미지 안에 들어갈 텍스트를 정확히 명시** — "상단 제목: 행사 기획 5단계", "라벨: 목표 설정, 예산 수립" 식으로
- **복잡한 도표(플로우차트, 의사결정 트리 등)는 이미지 대신 HTML/CSS로 구현** — 텍스트가 5개 이상 들어가는 복잡한 레이아웃은 이미지로 만들면 깨질 확률 매우 높음
- 한글 렌더링이 깨지면 → 텍스트를 더 줄여서 재생성

**좋은 예 (텍스트 적음, 깨지지 않음):**
- 제목 1개 + 라벨 3~5개 + 아이콘
- 파이차트 + 항목명만 (설명 없이)
- 비교 카드 + 제목 + 한 줄 키워드

**나쁜 예 (텍스트 많음, 100% 깨짐):**
- 의사결정 트리 + 각 노드에 2줄 설명
- 인포그래픽 + 4개 코너에 설명문
- 플로우차트 + 각 단계에 상세 설명

## 이미지 사양

| 항목 | 값 |
|------|-----|
| 해상도 | 1200 x 675px (16:9) 기본 |
| 썸네일 | 1200 x 675px |
| 인포그래픽 | 1200 x 800~1200px (세로 가변) |

### 세이프존 규칙 (썸네일 잘림 방지)

블로그 목록(3:2)과 SNS 공유에서 상하가 잘리므로:
- **핵심 텍스트/아이콘은 이미지 중앙 60% 영역에** 배치
- **상하 20%는 여백 또는 배경**으로 — 잘려도 내용 손실 없음
- 프롬프트에 명시: "핵심 요소를 중앙에 배치하고 상하 여백 충분히"
| 포맷 | WebP (quality 90) |
| 최대 크기 | 500KB 이하 |

### 업로드 경로

```
Supabase Storage > blog 버킷 > images/{timestamp}-{type}.webp

예: images/1710500000-infographic.webp
    images/1710500000-thumbnail.webp
```

### 업로드 방법

생성된 이미지를 다운로드 후 `/api/admin/blog/upload` 엔드포인트로 업로드:
```bash
curl -s -X POST http://localhost:4000/api/admin/blog/upload \
  -F "file=@/path/to/generated-image.webp" \
  | jq -r '.url'
```

## 품질 확인

1. 이미지 생성 후 **반드시 사용자에게 보여주고 확인** 받기
2. 텍스트가 깨지거나 읽기 어려우면 재생성
3. 브랜드 색상과 맞지 않으면 프롬프트 수정 후 재생성
4. 확인 완료된 이미지만 Storage에 업로드

## 공통 규칙

- Supabase Storage 버킷: `blog` / 경로: `images/{timestamp}-{type}.webp`
- 이미지 업로드: `/api/admin/blog/upload` (Sharp WebP quality 90, 최대 1200px)
- 나노바나나는 반드시 **Pro 모델(gemini-3-pro-image-preview)** 사용
- Flash 모델 사용 시 반드시 사용자 확인
- 포트폴리오 사진 재사용 금지 (홈페이지 중복)
- **작업 전 기존 구현 확인 필수** (CLAUDE.md 규칙)
