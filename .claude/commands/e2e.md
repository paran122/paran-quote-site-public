# E2E 테스트 생성기

너는 Playwright MCP를 사용하는 QA 전문가다.

## 기본 설정
- 베이스 URL: http://localhost:4000
- 테스트 대상: $ARGUMENTS

## 프로젝트 라우트
| 경로 | 설명 |
|------|------|
| `/` | 랜딩 페이지 |
| `/work` | 포트폴리오 목록 |
| `/work/[slug]` | 포트폴리오 상세 |
| `/blog` | 블로그 |
| `/blog/[slug]` | 블로그 상세 |
| `/faq` | FAQ |
| `/privacy` | 개인정보처리방침 |
| `/admin/*` | 관리자 |

## 테스트 진행 순서

### 1단계: 페이지 접속 및 스냅샷
- `browser_navigate`로 대상 페이지 접속
- `browser_snapshot`으로 페이지 구조 파악 (스크린샷보다 스냅샷 우선)
- 콘솔 에러 확인: `browser_console_messages` (level: "error")

### 2단계: 핵심 요소 검증
- 주요 텍스트, 버튼, 링크가 렌더링되는지 스냅샷으로 확인
- 이미지 깨짐 여부: `browser_evaluate`로 `naturalWidth === 0`인 img 체크
- 네트워크 에러: `browser_network_requests`로 실패한 요청 확인

### 3단계: 인터랙션 테스트
- `browser_click`으로 버튼/링크 클릭 후 결과 확인
- `browser_fill_form`으로 폼 입력 테스트
- `browser_snapshot`으로 변경된 상태 확인

### 4단계: 반응형 테스트
- `browser_resize`로 뷰포트 변경 후 레이아웃 확인
  - 모바일: 375 x 812
  - 태블릿: 768 x 1024
  - 데스크톱: 1280 x 720

### 5단계: 네비게이션 테스트
- GNB 메뉴 링크 클릭 → 올바른 페이지 이동 확인
- 뒤로가기(`browser_navigate_back`) 정상 동작 확인

## 테스트 항목 체크리스트
1. 페이지 로딩 및 주요 요소 렌더링
2. 콘솔 에러 없음
3. 네트워크 요청 실패 없음
4. 깨진 이미지 없음
5. 네비게이션 링크 정상 작동
6. 폼 입력 및 유효성 검사
7. 버튼 클릭 및 이벤트 동작
8. 반응형 레이아웃 (모바일/태블릿/데스크톱)
9. API 호출 정상 응답

## 실패 시 처리
- 실패 항목 발견 시 `browser_take_screenshot`으로 증거 캡처 (test-screenshots/ 폴더에 저장)
- 코드 수정이 필요하면 한 파일씩 수정
- 수정 후 해당 항목만 재테스트

## 출력 형식

### 테스트 결과 요약
| # | 항목 | 상태 | 비고 |
|---|------|------|------|
| 1 | 페이지 로딩 | PASS/FAIL | 세부 내용 |

### 실패 항목 상세 (있을 경우)
각 실패 항목마다:
- **문제**: 무엇이 실패했는지
- **원인**: 왜 실패했는지
- **스크린샷**: 캡처 경로
- **수정**: 어떻게 고쳤는지 (코드 변경 포함)

### 최종 판정
- 전체 PASS → "테스트 통과"
- FAIL 존재 → 수정 후 재테스트, 최종 결과 보고
